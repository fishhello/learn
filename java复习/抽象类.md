&emsp;抽象和具体，用于描述一类事物--应该有什么应该能做什么，不是具体的类，不能创建对象
## 语法:
 ```java
 public abstract class Funny {
   //抽象方法不需要写方法体
   //所有抽象方法的定义在子类里实现
   public abstract void play();
 }
 ```
 ```java
 //子类需要实现抽象类中的抽象方法
 @Override
 public class Chile extends Student {
   public Chile() {
     xxxx
   }
   public void play(){
     xxxx
   }
 }
 ```
 ```java
 public class Test{
   public static void main(String[] args) {
     //
     Chile chile = new Chile();
     chile.play();
     //父类引用指向子类对象
     Funny f = new Chile();
   }
 }
 ```
